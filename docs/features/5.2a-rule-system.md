# 5.2a. Hệ Thống Quy Tắc (Rule System)


**Status:** Implemented baseline / Next hardening  
**Owner docs:** `docs/features/5.2a-rule-system.md` (this file)  
**Code areas:** `server/pkg/models/rule.go`, `server/internal/service/rule.go`, `server/internal/orchestrator/prompt.go`, `server/internal/repository/rule.go`  
**Blocking decisions:** Final precedence model for global vs project rules and how conflicts are reported to users.  
**Acceptance criteria:** Global rules are immutable in agent system context, project rules are injected by task/project context, and conflicting local rules are rejected.

**Mục tiêu:** Kiểm soát hành vi của AI thông qua một kiến trúc ngữ cảnh phân lớp nghiêm ngặt (Strict Layered Context), đảm bảo tính nhất quán và an toàn.

---

#### A. Kiến trúc Ngữ cảnh Phân lớp (Strict Layered Context)

Hệ thống phân chia các quy tắc thành các lớp với mức độ ưu tiên khác nhau:

1.  **Quy tắc toàn cầu (Global Rules):**
    *   **Mô tả:** Các quy tắc cốt lõi về bảo mật và quản trị (ví dụ: không tiết lộ API key, luôn viết unit test, kiểm tra quyền truy cập).
    *   **Tính chất:** Bất biến (Immutable).
    *   **Áp dụng:** Được tiêm trực tiếp vào **System Prompt** của Agent ở cấp độ tổ chức (Organization) và không thể bị ghi đè bởi bất kỳ quy tắc cấp thấp nào.

2.  **Quy tắc dự án (Project Rules):**
    *   **Mô tả:** Các quy ước mã hóa, định dạng, và kiến trúc cụ thể của dự án (ví dụ: sử dụng Next.js App Router, kiến trúc Hexagonal, style TailwindCSS).
    *   **Tính chất:** Có thể tùy chỉnh cho từng dự án.
    *   **Áp dụng:** Được tiêm động vào **Task Context**. Hệ thống Orchestrator sẽ báo lỗi hoặc AI sẽ từ chối thực hiện nếu phát hiện xung đột với Global Rules.

3.  **Quy tắc theo Task (Task-specific Rules):**
    *   **Mô tả:** Các hướng dẫn và ràng buộc tạm thời cho một công việc cụ thể (ví dụ: "chỉ sửa logic trong file X, không thay đổi file Y").
    *   **Tính chất:** Tồn tại trong phạm vi của một Task.
    *   **Áp dụng:** Được cung cấp bởi người dùng hoặc Agent Planner khi tạo/phân tích Task.

#### B. Cơ chế Thực thi (Enforcement)

*   **Prompt Assembly:** Trước khi gửi request đến LLM, Orchestrator sẽ tổng hợp (assemble) các rules theo thứ tự ưu tiên: `Global Rules` + `Agent Role Constraints` + `Project Rules` + `Task Rules`.
*   **Conflict Detection:** Các Agent (đặc biệt là Reviewer Agent) được cấp quyền đánh giá xem code có vi phạm các quy tắc đã đề ra hay không và từ chối duyệt PR nếu vi phạm.
