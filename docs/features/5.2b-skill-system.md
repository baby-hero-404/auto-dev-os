# 5.2b. Hệ Thống Kỹ Năng (Skill System)


**Status:** Implemented baseline / Planned target  
**Owner docs:** `docs/features/5.2b-skill-system.md` (this file)  
**Code areas:** `server/pkg/models/skill.go`, `server/internal/service/skill.go`, `server/internal/repository/skill.go`, `web/src/` skills UI  
**Blocking decisions:** Whether to persist Git-backed CMS sync state in DB or rely fully on the file system.  
**Acceptance criteria:** (Baseline) DB-backed global skills CRUD works with prompt/tool metadata only. (Target) Executable skills run in isolated environments, dynamically loaded via JIT Loading (Librarian Protocol).

**Mục tiêu:** Quản lý và cung cấp các hành động/công cụ (tools/skills) cho Agent một cách an toàn, linh hoạt, và tối ưu hóa token.

---

#### A. Trạng thái hiện tại (Current Baseline)

*   **DB-backed Global Skills CRUD:** Các kỹ năng hiện tại được lưu trữ và cấu hình trực tiếp trong cơ sở dữ liệu (PostgreSQL). Admin có thể thực hiện CRUD qua API/UI.
*   **Seeding từ Prompt Base:** Cung cấp sẵn một tập hợp các kỹ năng cốt lõi (ví dụ: đọc/ghi file, clone repo, chạy test) được seed vào hệ thống từ bộ thư viện Prompt Base có sẵn.
*   **Kiểm thử mô phỏng (Dry-run):** Các kỹ năng hiện hành khi test qua API chỉ trả về kết quả mô phỏng (`dry_run: true`) chứ chưa thực thi mã trực tiếp.

#### B. Mục tiêu quy hoạch (Planned Target)

1.  **Hệ thống Quản lý Kỹ năng (Hybrid Git-backed CMS):**
    Hệ thống sẽ chuyển sang quản lý kết hợp giữa File-system based và Web UI để tối ưu hóa context token và dễ dàng quản lý:
    *   **Cấu trúc File-system based:** Các kỹ năng và prompt của AI được lưu trữ theo định dạng `SKILL.md` (hoặc tương đương), tổ chức gọn gàng theo thư mục.
    *   **Version Control qua Git:** Sử dụng Git để quản lý phiên bản minh bạch cho mọi thay đổi của kỹ năng.
    *   **Local Dashboard (Web UI):** Giao diện cho các thao tác CRUD. Khi thay đổi, backend tự động ghi đè hoặc xóa các file `.md` và cập nhật file index (`registry.json`).

2.  **Giao thức Thủ thư (Librarian Protocol / JIT Loading):**
    *   Thay vì gán trực tiếp kỹ năng cứng vào Agent, Agent Planner sẽ sử dụng file index (`registry.json` hoặc DB metadata) để phân tích ý định (Intent Analysis).
    *   Chỉ nạp động (Just-In-Time) những kỹ năng cần thiết nhất cho từng Task/Sub-task.
    *   Sau khi sử dụng xong, Agent giải phóng ngữ cảnh (context) để tránh quá tải token.

3.  **Cách ly thực thi (Sandboxing):**
    *   Đảm bảo mọi kỹ năng thực thi mã (code execution) phải chạy trong môi trường cô lập (Docker/SSH) để tránh rủi ro bảo mật thay vì chỉ dry-run.
