# 5.2b. Skill System (Simple Git-Sync System)

**Status:** Redesigning / Next Refactoring  
**Owner docs:** `docs/features/5.2b-skill-system.md` (this file)  
**Code areas:** `server/pkg/models/skill.go`, `server/internal/service/skill.go`, `server/internal/repository/skill.go`, `web/src/` skills UI  
**Acceptance criteria:**
- No UI-based custom skill creation/editing. Skills are completely read-only on the UI.
- Manage skills strictly via Git Repository sync (e.g., prompting base or custom git).
- The repo format must have a `registry.json` or `registry.min.json` manifest at its root.
- Admin inputs a Git Repo URL and triggers a Sync (Clone/Pull).
- UI supports exploring the synced skill repository's folder/file tree structure and reading file contents.

---

## A. Thiết Kế Giao Diện & Quản Lý Đơn Giản Hóa

Thay vì hỗ trợ CRUD tùy biến trực tiếp trên cơ sở dữ liệu hoặc giao diện (gây phức tạp, dễ xung đột phiên bản), hệ thống chuyển sang mô hình **Git-as-Source-of-Truth** hoàn toàn.

### 1. Luồng Hoạt Động (Workflow)
1. **Đăng ký Nguồn (Skill Sources):** Admin chỉ cần cung cấp URL của một Git repository chứa bộ skills (ví dụ: `https://github.com/baby-hero-404/prompt_base.git`).
2. **Đồng bộ hóa (Git Sync):**
   - Lần đầu: Hệ thống thực hiện `git clone` repository về thư mục cục bộ `skills_root/git/{repo_name}`.
   - Các lần tiếp theo: Khi admin nhấn nút **"Sync"**, backend thực hiện `git pull` để lấy cập nhật mới nhất.
3. **Phân tích Cấu Trúc (Manifest Registry):**
   - Thư mục gốc của repository phải có file cấu hình định danh `registry.json` hoặc `registry.min.json` (theo format của prompt_base).
   - Backend phân tích file registry này để hiển thị danh sách các Skill trên giao diện.
4. **Giao Diện Xem Chi Tiết (Folder/File Viewer):**
   - Loại bỏ hoàn toàn các nút Tạo (Create), Sửa (Edit), Xóa (Delete) hay dry-run skill tùy ý trên giao diện. Giao diện Skill hoàn toàn ở chế độ **Read-only**.
   - UI hỗ trợ xem danh sách Skill và duyệt cấu trúc thư mục dạng cây (Folder Tree) của các repository đã đồng bộ.
   - User có thể nhấp vào từng file trong thư mục của Skill (ví dụ: `SKILL.md`, các file script hoặc file cấu hình phụ trợ) để đọc nội dung trực tiếp trên Web UI.

## B. Quy Chuẩn Định Dạng Skill Repository

Một repository được coi là hợp lệ để đồng bộ nếu tuân thủ các quy tắc sau:

```
[git-repo-root]/
├── registry.json (hoặc registry.min.json)
└── skills/
    ├── core/
    │   └── architecture/
    │       ├── SKILL.md
    │       └── details.txt
    └── tech/
        └── database-design/
            └── SKILL.md
```

### registry.json Format:
```json
{
  "skills": {
    "core": [
      {
        "id": "architecture",
        "name": "Architecture Planner",
        "description": "Analyze system design, evaluate trade-offs, and design architectures.",
        "path": "skills/core/architecture/SKILL.md"
      }
    ]
  }
}
```

## C. Danh Sách APIs & UI Cần Cập Nhật

### 1. Backend Service (`server/internal/service/skill.go`)
- **Loại bỏ:** `Create`, `Update`, `Delete` đối với từng skill đơn lẻ.
- **Giữ lại & Cải tiến:** 
  - `ListSources`, `AddSource`, `DeleteSource`, `SyncSource` (Git clone/pull).
  - API duyệt cấu trúc thư mục của repository: `GET /api/skills/sources/{sourceID}/files?path=...` trả về danh sách thư mục con và file.
  - API đọc nội dung chi tiết file: `GET /api/skills/sources/{sourceID}/file-content?path=...` trả về text content của file.

### 2. Frontend UI (`web/src/app/skills/page.tsx`)
- Chỉ hiển thị ô nhập URL Repository Git và danh sách nguồn Git đã add kèm theo nút **Sync / Refresh**.
- Hiển thị danh sách các Skill lấy từ `registry.json` đã gộp.
- Khi click vào một Skill, hiển thị:
  - Một panel bên trái hiển thị cấu trúc file/folder của Skill đó (đọc qua API duyệt file).
  - Một panel bên phải hiển thị trình xem code/markdown (File Content Viewer) có syntax highlighting khi click chọn file tương ứng.
- Tuyệt đối không có form Tạo/Sửa skill.
