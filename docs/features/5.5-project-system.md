# 5.5. Hệ Thống Dự Án (Project System)


**Status:** Implemented baseline / Planned enhancements  
**Owner docs:** `docs/ARCHITECTURE.md`  
**Code areas:** `server/internal/service/project`, `server/internal/repository`, `server/pkg/models`, `web/src/` project screens  
**Blocking decisions:** Scope of shared knowledge base and project-level workflow defaults.  
**Acceptance criteria:** (Baseline) Project acts as an organizational unit containing repositories. (Target) Project contains shared configuration, rules/skills references, and AI workflow defaults.

**Khái niệm:** Một Project là một đơn vị tổ chức cấp cao, hiện tại đóng vai trò là nơi nhóm các repository lại với nhau. Trong tương lai (Planned Expansion), Project sẽ mở rộng để quản lý cấu hình workflow AI chung, và các quy tắc/kiến thức chia sẻ để tạo nên một môi trường phát triển thống nhất.

---

#### A. Cấu trúc Entity

**1. Baseline Hiện tại (Current Baseline):**
```
Project
 ├── Repositories[]         — Git repos liên kết (xem §5.4)
```

**2. Mở rộng theo Kế hoạch (Planned Expansion):**
```
Project
 ├── Members[]              — Thành viên được gán vai trò (vai trò được định nghĩa ở cấp Organization)
 ├── Rules[]                — Project-level rules (injected vào Task Context, xem §5.2a)
 ├── Skills[]               — Tham chiếu skill sets ưu tiên cho project (xem §5.2b)
 ├── Knowledge Base         — Tài liệu kiến trúc, RFC, coding conventions
 ├── AI Workflow Defaults   — Model Level mặc định, auto-review policy
 └── Environments[]         — Dev, Staging, Production configs
```

#### B. Luồng Vận Hành (Planned Operation)

```
Bước 1: Tạo Project
        ├── Nhập tên, mô tả, ngôn ngữ chính
        └── Chọn template (blank / web-app / microservice / monorepo)

Bước 2: Cấu hình Git
        ├── Gắn Repository (xem §5.4)
        └── Chọn Git Account + default branch

Bước 3: Cấu hình AI Defaults
        ├── Default Model Level Group (Fast / Balanced / Powerful)
        ├── Default Autonomy Level (autonomous / supervised / approval_required)
        └── Auto-review policy (Easy: skip spec_review, Medium/Hard: required)

Bước 4: Thiết lập Knowledge Base
        ├── Upload hoặc liên kết tài liệu kiến trúc
        ├── Coding conventions và style guides
        └── RFC / ADR (Architecture Decision Records)

Bước 5: Quản lý thành viên (Project-level: chỉ thêm/bớt member)
        ├── Mời thành viên qua email hoặc username
        └── Gán vai trò từ danh sách vai trò đã định nghĩa ở cấp Organization
            (Tạo/sửa/xóa vai trò → Organization Settings)
```

#### C. AI Workflow Defaults (Planned)

Mỗi Project có thể cấu hình các giá trị mặc định cho workflow AI, áp dụng cho tất cả task trong project (trừ khi task override riêng):

| Setting | Giá trị mặc định | Mô tả |
| :------ | :---------------- | :----- |
| `default_model_level` | `balanced` | Model Level Group mặc định cho Agent Planner chọn |
| `default_autonomy` | `supervised` | Mức tự chủ mặc định của Agent |
| `auto_review_policy` | `complexity_based` | Easy → auto-skip `spec_review`; Medium/Hard → require human review |
| `max_retries` | `3` | Số lần retry tối đa trước khi task chuyển sang `failed` |
| `default_branch` | `main` | Branch mặc định để tạo PR target |

#### D. Knowledge Base (Planned)

Kho kiến thức chia sẻ giúp Agent hiểu rõ ngữ cảnh dự án khi thực thi task:

*   **Tài liệu kiến trúc:** System design, module dependencies, database schema — được inject vào Agent context khi phân tích task.
*   **Coding Conventions:** Style guides, naming conventions, patterns bắt buộc — Agent sử dụng làm tham chiếu khi viết code.
*   **RFC / ADR:** Các quyết định kiến trúc đã được thông qua — giúp Agent hiểu lý do đằng sau các design choices.
*   **Tích hợp RAG (Planned):** Sử dụng vector search (pgvector) để Agent tự tìm tài liệu liên quan thay vì inject toàn bộ vào context.
