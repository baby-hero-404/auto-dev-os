# 5.6. Hệ Thống Task


**Status:** Implemented baseline / Planned integrations  
**Owner docs:** `docs/ARCHITECTURE.md`  
**Code areas:** `server/internal/service/task`, `server/internal/repository`, `server/pkg/models`, `web/src/` task screens  
**Blocking decisions:** External issue tracker priority: Jira, Linear, GitHub Issues, Notion.  
**Acceptance criteria:** Developer can create tasks with repository context, task lifecycle states are persisted, and orchestrator can advance task status through planning/coding/review/testing/human review.

**Mục tiêu:** Cung cấp cơ chế để nhà phát triển tạo tác vụ và giao cho AI thực thi một cách tự động.

**Tính năng chính:**
*   **Tạo Task (Step 1):** Con người tạo tác vụ mới với yêu cầu chi tiết.
*   **Phân tích Task (Step 2):** Agent Planner sử dụng chuẩn OpenSpec để phân tích, chia nhỏ thành các task con (nếu cần khi task phức tạp). Đồng thời định nghĩa workflow cần thiết, chọn model level (Fast/Balanced/Powerful), và xác định vùng skill liên quan cần sử dụng.
*   **Review Plan (Step 3):** Con người xem xét và phê duyệt plan/spec đối với task phức tạp trước khi thực thi.
*   **Thực thi & PR (Step 4):** Các Agent chạy workflow, tự động hoàn thành sub-task và tạo Pull Request.
*   **Duyệt Code (Step 5):** Con người review và phê duyệt đoạn code cuối cùng.

**Vòng đời Task:** Quản lý trạng thái qua 3 Phase, sử dụng 10 status (khớp với `server/pkg/models/task.go`):

```
todo → analyzing → spec_review → coding ⟷ reviewing ⟷ fixing → testing → human_review → merged
                                                                                        ↘ failed (bất kỳ lúc nào)
```

| Phase | Status (DB value) | Mô tả | Workflow Step |
| :---- | :---------------- | :---- | :------------ |
| 📋 **Planning** | `todo` | Task vừa được tạo, chưa xử lý | Step 1 |
| | `analyzing` | Agent Planner đang phân tích, tạo spec, chia sub-tasks, chọn model level & skills | Step 2 |
| | `spec_review` | Con người review spec/plan (Medium/Hard). Easy → auto-skip | Step 3 |
| ⚡ **Executing** | `coding` | Agent đang viết code | Step 4 |
| | `reviewing` | Agent Reviewer đang review chéo code | Step 4 |
| | `fixing` | Agent đang fix lỗi từ review hoặc test | Step 4 |
| | `testing` | Agent đang chạy test | Step 4 |
| ✅ **Approval** | `human_review` | Con người review PR cuối cùng | Step 5 |
| | `merged` | Hoàn tất, code đã merge | Done |
| ❌ **Terminal** | `failed` | Lỗi xảy ra ở bất kỳ bước nào | — |

> **Deprecated statuses** (sẽ backfill migration): `assigned`, `planning`, `in_progress`, `completed`.

**Tích hợp trong tương lai:**
*   Tích hợp với các hệ thống quản lý tác vụ phổ biến như Jira.
*   Tích hợp với Linear để quản lý tác vụ hiệu quả.
*   Đồng bộ hóa với GitHub Issues.
*   Tích hợp với Notion để quản lý tài liệu và tác vụ.

**Dự án tham khảo:**
| Dự án       | Lý do tham khảo                                |
| :---------- | :--------------------------------------------- |
| Plane       | Trình quản lý issue hiện đại và thân thiện người dùng |
| OpenProject | Cung cấp workflow quản lý dự án cấp doanh nghiệp |
| Linear      | Nguồn cảm hứng về trải nghiệm người dùng (UX) trong quản lý tác vụ |
