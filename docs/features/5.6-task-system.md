**Mục tiêu:** Cung cấp cơ chế để nhà phát triển mô tả yêu cầu (task) và giao cho AI thực thi tự động. Task là đơn vị công việc cốt lõi — mọi thứ trong hệ thống đều xoay quanh task.

---

## Cách Task Hoạt Động (5 Bước)

```
Bước 1: Tạo Task
   Con người tạo task mới với mô tả chi tiết yêu cầu.

Bước 2: AI Phân Tích
   Agent Planner phân tích task → chia thành sub-tasks nếu cần → tạo spec theo chuẩn
   OpenSpec → chọn model level (Fast/Balanced/Powerful) → xác định skill cần thiết.

Bước 3: Con Người Review Plan
   Task phức tạp (Medium/Hard): con người xem xét và phê duyệt plan trước khi thực thi.
   Task đơn giản (Easy): tự động skip bước này.

Bước 4: AI Thực Thi & Tạo PR
   Agent viết code → Agent khác review chéo → sửa lỗi nếu có → chạy test → tạo Pull Request.

Bước 5: Con Người Duyệt Code
   Con người review PR cuối cùng → Approve → Code được merge.
```

## Vòng Đời Task (10 Trạng Thái)

```
todo → analyzing → spec_review → coding ⟷ reviewing ⟷ fixing → testing → human_review → merged
                                                                                        ↘ failed
```

| Phase | Status | Mô tả |
|:------|:-------|:------|
| 📋 **Planning** | `todo` | Task vừa được tạo, chưa xử lý |
| | `analyzing` | Agent Planner đang phân tích, tạo spec, chia sub-tasks |
| | `spec_review` | Con người review spec/plan. Easy → auto-skip |
| ⚡ **Executing** | `coding` | Agent đang viết code |
| | `reviewing` | Agent Reviewer đang review chéo code |
| | `fixing` | Agent đang sửa lỗi từ review |
| | `testing` | Agent đang chạy test |
| ✅ **Approval** | `human_review` | Con người review PR cuối cùng |
| | `merged` | Hoàn tất — code đã merge |
| ❌ **Terminal** | `failed` | Lỗi xảy ra ở bất kỳ bước nào |

> Task ở trạng thái `failed` có thể được con người reset về `todo` để chạy lại.

## Cơ Chế Multi-Repo Workspace

Hệ thống Task hỗ trợ linh hoạt việc thực thi trên một hay nhiều Codebase (Multi-Repo):

- **Single-Repo Task (Mặc định):** Khi một Task được gắn với một `RepositoryID` cụ thể, AI Sandbox chỉ chứa mã nguồn của Repository đó. Quá trình kiểm tra lỗi (`git diff`), tạo branch và Pull Request chỉ diễn ra cục bộ trong scope của Repo mục tiêu.
- **Multi-Repo / Cross-Repo Task:** Khi một Task không gắn `RepositoryID`, nó có scope là toàn bộ Project. Orchestrator sẽ tự động clone tất cả Repositories của Project vào các thư mục con bên trong Sandbox. Khi Task hoàn thành, Orchestrator quét thay đổi trên từng thư mục con, tiến hành tạo Pull Request độc lập cho **mỗi Repository có chứa thay đổi**. Một Task có thể sinh ra nhiều `pr_urls` và theo dõi chúng đồng thời.
