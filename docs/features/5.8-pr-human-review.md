# 5.8. PR & Human Review


**Status:** Implemented baseline / Planned assistant enhancements  
**Owner docs:** `docs/ARCHITECTURE.md`  
**Code areas:** `server/internal/orchestrator/pr_generator`, `server/internal/handler/pr`, `server/internal/service/task`, `web/src/` PR/review UI  
**Blocking decisions:** How much AI explanation should be generated automatically vs on reviewer request.  
**Acceptance criteria:** System creates PRs, persists PR metadata, routes tasks to human review, and supports reviewer approval before merge.

**Mục tiêu:** Tự động hóa quy trình tạo Pull Request và cung cấp công cụ hỗ trợ con người review code hiệu quả, đảm bảo chất lượng mã nguồn trước khi merge.

---

#### A. Luồng Vận Hành PR

```
1. Agent hoàn thành coding + test → tạo PR tự động
   ├── Branch: autocode/task-{task_id}
   ├── Tiêu đề: "AutoCodeOS: {task.title}"
   ├── Body gồm:
   │   ├── Task ID + description
   │   ├── Danh sách file thay đổi (added / modified / deleted)
   │   ├── Risk assessment (low / medium / high)
   │   └── Test results summary
   ├── Labels: auto-generated (risk-level, task-type, agent-role)
   └── Task status → `human_review`

2. Reviewer nhận thông báo (Notification - Planned)
   ├── Xem PR diff + AI-generated summary (Baseline)
   ├── Hỏi AI PR Assistant nếu cần context thêm (Planned)
   │   └── Ví dụ: "Tại sao thay đổi logic này?" → AI giải thích dựa trên task spec + code context
   └── Quyết định:
       ├── ✅ Approve → Task status → `merged` (Baseline)
       │   └── Auto-merge nếu review policy cho phép (Easy task) (Planned)
       └── ❌ Request Changes → Task status quay lại `fixing`
           → Agent nhận feedback từ reviewer
           → Sửa code, chạy lại test, push commit mới
           → Task status: `fixing` → `reviewing` → `testing` → `human_review`
           → Vòng lặp tiếp tục cho đến khi approve
```

#### B. Review Policy theo Task Complexity

*   **Current Baseline:** Duyệt/từ chối PR thủ công, với PR summary, risk, và task status transitions cơ bản.
*   **Planned Target:** 
| Complexity | Review Type | Số Reviewer tối thiểu | Auto-merge sau Approve |
| :--------- | :---------- | :-------------------- | :--------------------- |
| Easy       | Lightweight | 1                     | Có |
| Medium     | Standard    | 1                     | Không (merge thủ công) |
| Hard       | Deep Review | 2 (cross-review)      | Không |

#### C. PR Template Structure

Mỗi PR được tạo tự động sẽ tuân theo template chuẩn:

```markdown
## AutoCodeOS: {task.title}

**Task:** #{task_id}
**Agent:** {agent.role} ({model_level})
**Complexity:** {task.complexity}

### Summary
{AI-generated tóm tắt thay đổi}

### Changes
| File | Action | Lines Changed |
|------|--------|---------------|
| ... | Modified | +12 / -3 |

### Risk Assessment
- **Level:** {low/medium/high}
- **Reason:** {giải thích lý do đánh giá risk}

### Test Results
- ✅ Unit tests: {passed}/{total}
- ✅ Lint: passed
- ⚠️ Integration tests: {status}
```

#### D. AI PR Assistant (Planned Target)

*   **On-demand explanation:** Reviewer có thể hỏi AI về bất kỳ đoạn code nào trong PR — AI trả lời dựa trên task spec, codebase context, và lịch sử thay đổi.
*   **Risk highlighting:** AI tự động đánh dấu các thay đổi có risk cao (ví dụ: sửa logic xử lý thanh toán, thay đổi schema DB, xóa file).
*   **Suggestion mode:** AI có thể đề xuất cải thiện code nhưng không tự động áp dụng — quyết định thuộc về reviewer.

#### E. Chính sách Merge (Merge Policy)

*   **Current Baseline:** Reviewer có quyền quyết định Approve và Merge thủ công qua UI.
*   **Planned Target:**
    *   Code chỉ được merge khi tất cả test pass.
    *   Có ít nhất số lượng reviewer tối thiểu approve (theo complexity).
    *   Không có unresolved review comments.
    *   PR không conflict với target branch.
