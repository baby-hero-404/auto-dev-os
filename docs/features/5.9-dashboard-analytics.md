# 5.9. Dashboard & Analytics


**Status:** Implemented baseline / Planned observability expansion  
**Owner docs:** `docs/ARCHITECTURE.md`; `docs/features/5.1-unified-ai-gateway.md` for gateway telemetry  
**Code areas:** `server/internal/handler/analytics_dashboard`, `server/internal/service/analytics_dashboard`, `server/internal/handler/audit`, `web/src/` dashboard screens  
**Blocking decisions:** Langfuse/Helicone/OpenObserve integration depth vs custom telemetry only.  
**Acceptance criteria:** Dashboard shows project/task/agent status and key metrics such as success rate, retries, token usage, cost, latency, and failure reasons (via `workflow_jobs.last_error`).

**Mục tiêu:** Cung cấp giao diện tổng quan trực quan để theo dõi hiệu suất hệ thống, trạng thái task/agent, và chi phí sử dụng AI — giúp quản lý và tối ưu hóa quy trình phát triển.

---

#### A. Dashboard Sections

| Section | Nội dung hiển thị | Data Source |
| :------ | :----------------- | :---------- |
| **Overview** | Active tasks, Open PRs, Failed runs, Agent status summary | `tasks`, `workflows`, `agents` |
| **Task Analytics** | Success rate, Avg completion time, Status distribution chart | `tasks` aggregate |
| **Agent Performance** | Per-agent: success rate, retries, tokens used, cost, avg completion time | `workflow_jobs`, `token_usage` |
| **Gateway Usage** | Requests/day, Token consumption by provider, Cost breakdown, Latency P50/P95 | `token_usage` |
| **Audit Trail** | Chronological event log: user actions, agent actions, key lifecycle events | `audit_logs` |


#### B. Key Metrics

| Metric | Công thức | Ý nghĩa |
| :----- | :-------- | :------- |
| **Task Success Rate** | `merged / (merged + failed)` × 100% | Tỷ lệ task hoàn thành thành công |
| **Avg Cycle Time** | `AVG(tasks.updated_at - tasks.created_at)` | Thời gian trung bình từ tạo task đến hoàn thành |
| **Token Efficiency** | `total_tokens / tasks_completed` | Hiệu quả sử dụng token trên mỗi task |
| **Cost per Task** | `total_cost / tasks_completed` | Chi phí trung bình mỗi task |
| **Agent Retry Rate** | `retries / total_attempts` × 100% | Tỷ lệ phải retry — cao = agent hoặc prompt cần cải thiện |
| **Provider Uptime** | `successful_requests / total_requests` × 100% | Độ ổn định của từng provider (liên quan multi-key fallback) |

#### C. Project Dashboard (Chi tiết)

Trang tổng quan cấp project, hiển thị:
*   **Task Board:** Phân bổ task theo status (todo → analyzing → coding → ... → merged/failed) dạng kanban hoặc bar chart.
*   **Active Agents:** Danh sách agent đang chạy, task hiện tại, model level đang sử dụng.
*   **Open PRs:** Danh sách PR chờ review với risk level và thời gian chờ.
*   **Recent Failures:** Top 5 task failed gần nhất với failure reason và link đến log chi tiết.
*   **Cost Summary:** Tổng chi phí trong khoảng thời gian (ngày/tuần/tháng), breakdown theo provider và model level.

#### D. Agent Metrics (Chi tiết)

Trang phân tích hiệu suất từng agent:
*   **Performance Over Time:** Biểu đồ line chart theo thời gian: success rate, avg cycle time, tokens per task.
*   **Task History:** Lịch sử task đã xử lý với kết quả (merged/failed), thời gian hoàn thành, số lần retry.
*   **Cost Tracking:** Chi phí tích lũy theo agent, so sánh giữa các agent cùng role.
*   **Skill Usage (Planned):** Thống kê các skill được JIT-load nhiều nhất cho agent này.

#### E. Audit Log Structure

Mỗi audit event ghi nhận đầy đủ thông tin:

| Field | Mô tả | Ví dụ |
| :---- | :----- | :---- |
| `timestamp` | Thời điểm xảy ra | `2026-06-17T11:00:00Z` |
| `actor_type` | Loại tác nhân | `user` / `agent` / `system` |
| `actor_id` | ID tác nhân | `user-123` / `agent-backend-01` |
| `action` | Hành động thực hiện | `task.created`, `pr.approved`, `credential.rotated` |
| `resource_type` | Loại resource bị ảnh hưởng | `task`, `agent`, `provider_credential` |
| `resource_id` | ID resource | `task-456` |
| `details` | Metadata bổ sung (JSON) | `{"old_status": "coding", "new_status": "testing"}` |

#### F. Mở rộng trong tương lai

*   **Langfuse/Helicone integration:** Tích hợp observability platform chuyên dụng cho LLM — trace từng request, prompt debugging, cost optimization.
*   **Custom Alerts:** Cảnh báo khi metric vượt ngưỡng (ví dụ: cost/day > $50, failure rate > 30%).
*   **Export & Reporting:** Xuất báo cáo dạng CSV/PDF cho management review.
*   **Comparative Analytics:** So sánh hiệu suất giữa các model level, provider, hoặc agent role.

**Dự án tham khảo:**
| Dự án         | Lý do tham khảo                                |
| :------------ | :--------------------------------------------- |
| Langfuse      | Khả năng quan sát AI (AI observability) toàn diện |
| Helicone      | Nền tảng phân tích và tối ưu hóa LLM          |
| OpenObserve   | Ý tưởng về hệ thống logging và dashboard linh hoạt |
